## Para saber mais: o que é JSON?

**JSON**, ou **JavaScript Object Notation**, é uma **notação para objetos** e **amplamente utilizado para a troca de dados entre sistemas**, especialmente em APIs RESTful, como a que estamos utilizando no nosso projeto. O nome "notação para objetos" pode assustar um pouco, mas não se trata de nada além de **pegar objetos que temos no nosso contexto de programação**, ou seja, instâncias de uma classe, **e convertê-los para um documento textual bem formatado**.

Embora o nome sugira uma ligação com o JavaScript, o **JSON é independente de linguagem** e pode ser **utilizado por diversas linguagens** de programação, incluindo Dart, Python, Java, entre outras. Inclusive essa é a beleza de um JSON, eu posso pegar um objeto que está em uma classe Dart, **convertê-lo para uma `String` usando a notação JSON**, **enviar pela internet**, e essa `String` pode **ser recebida por um servidor rodando Python, Golang, JS, PHP, ou qualquer outra**, que ele poderá "decodificar" essa `String` em um objeto equivalente (com os mesmos dados) que faça sentido na outra linguagem.

Por isso essa notação é tão usada hoje em dia no contexto de APIs: o JSON é o **formato mais comum para transmitir dados entre o cliente** (como um aplicativo ou site) **e o servidor**. Essa popularidade se deve à sua simplicidade e legibilidade, que tornam o JSON ideal para a comunicação entre diferentes sistemas de forma eficiente e organizada.

O JSON não é usado apenas para comunicação na web. Existem **várias outras aplicabilidades**, como por exemplo, muitos sistemas usam arquivos JSON para **armazenar localmente configurações que precisam ser lidas pela aplicação**.

### Estrutura básica

Um arquivo ou string JSON lembrará muito a estrutura de um `Map` ou de uma `List` (ou ambos), consistindo em:

- **Objetos**: representados por `{}`. Dentro das chaves, **temos pares de chave-valor**. Por exemplo:

```json
{
  "nome": "Ana",
  "idade": 25,
  "cidade": "São Paulo"
}
```

Aqui, `"nome"`, `"idade"` e `"cidade"` são as chaves, e `"Ana"`, `25`, `"São Paulo"` são seus respectivos valores. Esses objetos em geral serão tipos primitivos, como `Strings`, números inteiros, números fracionados, booleanos, `DateTime` ou timestamps, nulos e listas.

- **Arrays (listas)**: representados por `[]`. Um array é uma **coleção de valores ou objetos**:

```json
[
  {"nome": "Ana", "idade": 25},
  {"nome": "Carlos", "idade": 30}
]
```

Neste exemplo, temos uma **lista de objetos**, cada um representando uma pessoa.
